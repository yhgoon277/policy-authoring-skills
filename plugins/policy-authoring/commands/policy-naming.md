# Policy Naming Readability

Use this command when the user wants to make policy names, sub-function names, or descriptions easier for non-technical planners to understand.

## Arguments

The user invoked this command with: $ARGUMENTS

## Instructions

1. Read `skills/policy-naming-readability/SKILL.md`.
2. Preserve item count, order, IDs, and core numeric values.
3. Rewrite only the wording unless the user explicitly asks for structure changes.
4. Use "name (ID)" rendering guidance where lists or tables are involved.

## Example Usage

```text
/policy-naming 세부기능 이름을 기획자가 이해하기 쉽게 다듬어줘
/policy-authoring:policy-naming 이 설명을 쉬운 말로 풀어써줘
```
